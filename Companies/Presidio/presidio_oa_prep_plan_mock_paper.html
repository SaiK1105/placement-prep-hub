Section C

-- C1
SELECT MAX(salary) FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);
-- or:
SELECT DISTINCT salary FROM employees ORDER BY salary DESC LIMIT 1 OFFSET 1;

-- C2
SELECT d.dept_name, AVG(e.salary) AS avg_sal
FROM employees e JOIN departments d ON e.dept_id = d.dept_id
GROUP BY d.dept_name
HAVING AVG(e.salary) > 50000
ORDER BY avg_sal DESC;

Section D

# D1 — O(n) time, O(1) space
def pivot_index(a):
    total, left = sum(a), 0
    for i, v in enumerate(a):
        if left == total - left - v:
            return i
        left += v
    return -1

# D2
from collections import Counter
def first_unique(s):
    c = Counter(s)
    for ch in s:
        if c[ch] == 1:
            return ch
    return '_'

Section E

MySQL. Money movement needs ACID transactions (debit + credit commit together or not at all), strong consistency (never a stale balance), a stable schema, and foreign-key integrity. NoSQL's eventual consistency suits logs/analytics at scale — fine as a side store — but the core ledger belongs in an RDBMS. Bonus maturity point: "MySQL for the ledger, NoSQL for logs/analytics."
