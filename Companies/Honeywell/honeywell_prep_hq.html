рЯУЦ The 7 things tomorrow tests

Each ends with a "prove it" вАФ if you can't answer it cold, that's your gap.

1 ¬Ј Arrays & strings patterns (the coding section's bread and butter)

  • Single-pass counter: max consecutive 1s, streaks, run-length. Reset on break, track best. O(n).
  • Two pointers: pair-sum in sorted array, reverse, palindrome check. Move the pointer that helps.
  • Sliding window: longest substring/subarray under a constraint вАФ grow right, shrink left while invalid.
  • Hashing: frequency maps for anagrams, first-non-repeating, duplicates. collections.Counter is your friend.
  • Prefix sums: range sums, equilibrium index.
Prove it: longest streak of 1s in [1,1,0,1,1,1,0,1] вАФ what do you track per step, and what's the answer? (best=3; track current run + best, reset current on 0)

2 ¬Ј Raw stdin I/O in Python (HirePro is NOT LeetCode)

No function signatures вАФ you read stdin, print stdout. Memorize this skeleton: 
import sys
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))
# ... solve ...
print(answer)
Multiple test cases usually come as t = int(input()) then a loop. Print exactly what's asked вАФ no labels like "Answer:", no extra spaces.
Prove it: input is "3 1 2 3" (n, then n ints). Write the two read lines from memory.

3 ¬Ј OOPS rapid theory

  • 4 pillars: encapsulation (bundle data+methods, restrict access), abstraction (hide implementation, expose interface), inheritance, polymorphism.
  • Overloading = same name, different params, resolved at compile time. Overriding = subclass redefines, resolved at runtime (virtual dispatch).
  • Abstract class: can hold constructors + instance state; interface (Java): contract only. Python does multiple inheritance (MRO); Java classes don't.
Prove it: which pillar does private + getters/setters implement, and which one is "runtime polymorphism"?

4 ¬Ј OS essentials

  • Process = own address space; threads share code/heap/globals but have own stack + registers.
  • Scheduling: FCFS, SJF (starvation risk!), Round Robin (time quantum), Priority (starvation вЖТ fix with aging).
  • Deadlock's 4 conditions: mutual exclusion, hold & wait, no preemption, circular wait.
  • Virtual memory: page fault = page not in RAM; too many = thrashing.
Prove it: name the 4 deadlock conditions in under 10 seconds.

5 ¬Ј DBMS essentials

  • Normalization: 1NF atomic values вЖТ 2NF no partial dependency вЖТ 3NF no transitive dependency.
  • ACID: Atomicity, Consistency, Isolation (concurrent txns don't see each other's intermediate state), Durability.
  • Index: faster SELECT, slower INSERT/UPDATE/DELETE. Primary key = UNIQUE + NOT NULL.
  • Joins: INNER = matches only; LEFT = all left rows, NULLs where unmatched. GROUP BY + HAVING filters groups.
Prove it: what exactly does 2NF eliminate, and on what kind of key does that situation arise? (partial dependency вАФ composite primary key)

6 ¬Ј Pseudocode / output prediction technique

  • Trace on paper, always. Make a variable table; update per iteration. Mental tracing is where marks die.
  • C increments: b = a++ вЖТ b gets old value; b = ++a вЖТ b gets new value.
  • Loop off-by-ones: range(5) = 0..4; while i < 10: i *= 2 from 1 hits 1,2,4,8.
  • Recursion: unwind to base case, then roll back up.
Prove it: f(n) = f(nвИТ1) + 2, f(0) = 0. What's f(4)? (8)

7 ¬Ј Aptitude speed math

  • Successive % change: +a% then вИТa% вЖТ net вИТa¬≤/100 % (e.g. +20 then вИТ20 = вИТ4%).
  • Time & work: rates add. A in 12d + B in 18d вЖТ 1/12 + 1/18 = 5/36 вЖТ 7.2 days.
  • Trains: km/h √Ч 5/18 = m/s. Passing a pole: time = length √Ј speed.
  • Average speed over equal distances = harmonic mean: 2xy/(x+y).
  • Dice/coins: count favorable √Ј total (36 for two dice, 2вБњ for n coins).
Prove it: 60 km at 30 km/h then 60 km at 60 km/h вАФ average speed? (40, not 45)

Q1 ¬Ј Sensor Uptime Streak (easy вАФ pattern of the reported "max consecutive 1s")

A machine reports its status every minute: 1 = online, 0 = offline. Given the day's log, print the length of the longest continuous online streak.

Input:  line 1: n (1 вЙ§ n вЙ§ 10вБµ)
        line 2: n space-separated integers (each 0 or 1)
Output: one integer вАФ longest run of 1s

Sample in:   8
             1 1 0 1 1 1 0 1
Sample out:  3
HintTrack cur and best. On 1: cur += 1, best = max(best, cur). On 0: cur = 0. One pass, O(n), O(1) space.
Solution + the twist to expect 
import sys
input = sys.stdin.readline
n = int(input())
a = list(map(int, input().split()))
cur = best = 0
for x in a:
    cur = cur + 1 if x == 1 else 0
    best = max(best, cur)
print(best)
Likely twist: "you may flip at most k zeros" вЖТ sliding window: grow right, count zeros in window, shrink left while zeros > k; answer = max window size.

Q2 ¬Ј Third Largest Alert Code (easy-medium вАФ the reported "third largest string" verbatim pattern)

Alerts arrive as strings. Print the third largest distinct string in lexicographic (dictionary) order. If fewer than 3 distinct strings exist, print NONE.

Input:  line 1: n (1 вЙ§ n вЙ§ 10вБі)
        next n lines: one lowercase string each
Output: the third largest distinct string, or NONE

Sample in:   5
             fire
             smoke
             gas
             fire
             heat
Sample out:  gas   (distinct sorted desc: smoke, heat, gas, fire)
Hintset() to dedupe вЖТ sorted(..., reverse=True) вЖТ index 2 if length вЙ• 3. Watch the trap: duplicates must not count twice.
Solution 
import sys
data = sys.stdin.read().split()
n = int(data[0])
s = sorted(set(data[1:n+1]), reverse=True)
print(s[2] if len(s) >= 3 else "NONE")

Q3 ¬Ј Non-Adjacent Sensor Load (medium вАФ easy-DP stretch, house-robber pattern)

n sensors sit in a row; sensor i draws p[i] watts when enabled. Adjacent sensors interfere, so no two neighbours may both be enabled. Print the maximum total wattage achievable.

Input:  line 1: n (1 вЙ§ n вЙ§ 10вБµ)
        line 2: n non-negative integers p[i] вЙ§ 10вБі
Output: one integer вАФ max sum of non-adjacent elements

Sample in:   5
             3 2 7 10 1
Sample out:  13   (3 + 10, or 3+7+1=11 < 13)
HintFor each i: either skip it (best so far) or take it (p[i] + best up to iвИТ2). Two rolling variables, no array needed.
Solution 
import sys
input = sys.stdin.readline
n = int(input())
p = list(map(int, input().split()))
take = skip = 0          # best ending with/without current
for x in p:
    take, skip = skip + x, max(skip, take)
print(max(take, skip))
