1 · Math — Digital Root

Sum the digits of N until one digit remains. 9875 → 29 → 11 → 2. N up to 10¹⁸.

Solution + explanation 
# loop version — passes
def digital_root(n):
    while n >= 10:
        n = sum(int(d) for d in str(n))
    return n
# O(1): 1 + (n-1) % 9  ("casting out nines" — digit sums preserve mod 9)

Mention the mod-9 shortcut in a comment — free sophistication points.

2 · Hashing + Heap — Top-K Frequent (reported "dictionary + heap" problem)

Given a list of product IDs, return the k most frequent, most-frequent first (ties: smaller ID first). [1,1,2,2,2,3], k=2 → [2,1].

Solution + explanation 
from collections import Counter
import heapq
def top_k(items, k):
    freq = Counter(items)                     # dictionary pass: O(n)
    return heapq.nsmallest(k, freq, key=lambda x: (-freq[x], x))

The reported problem class: count with a dict, select with a heap. nsmallest on (-count, id) handles both orderings in one line — O(n log k), better than full sort when k ≪ n. Drill cousin: LC 347 Top K Frequent Elements.

3 · Two Pointers — Max Water Between Two Buildings (reported)

Heights of buildings; rain fills the gaps between them. Return total trapped water. [3,0,2,0,4] → 7.

Solution + explanation 
def trapped(h):
    l, r = 0, len(h) - 1
    lmax = rmax = ans = 0
    while l < r:
        if h[l] < h[r]:
            lmax = max(lmax, h[l]); ans += lmax - h[l]; l += 1
        else:
            rmax = max(rmax, h[r]); ans += rmax - h[r]; r -= 1
    return ans

Water above cell i = min(maxLeft, maxRight) − h[i]. Two pointers: whichever side is lower is bounded by its own max — safe to settle it and move inward. O(n) time, O(1) space. This exact problem was reported in a Tredence OA (GfG writeup). Drill: LC 42 Trapping Rain Water.

4 · Arrays — Best Sale Window (max subarray)

Max sum of any contiguous subarray. [2,-8,3,-2,4,-10] → 5. Handle all-negative.

Solution + explanation 
def max_subarray(a):
    best = cur = a[0]
    for x in a[1:]:
        cur = max(x, cur + x)   # extend the run, or restart here
        best = max(best, cur)
    return best

Kadane's: at each element, "is the running sum helping (cur+x) or hurting (restart at x)?" Starting at a[0], not 0, makes all-negative arrays correct. Brute O(n²) first for partials, then this.

5 · Trees — Level Sum

Return the sum of nodes at each level. Tree 1(2, 3(4,5)) → [1, 5, 9].

Solution + explanation 
from collections import deque
def level_sums(root):
    if not root: return []
    out, q = [], deque([root])
    while q:
        out.append(sum(n.val for n in q))
        q = deque(c for n in q for c in (n.left, n.right) if c)
    return out

Level-batch BFS: sum the whole queue (one level), rebuild from children. Same skeleton solves level-order print, max-per-level, zigzag — one pattern, four questions.
