P1 - HIGH. This is Swym's #1 real-world ticket, and their documented interview exercise is built on it.

A merchant reports the wishlist button vanished right after they updated their theme. The most likely cause is:

  1. Swym's servers are down
  2. The theme update replaced the theme's files, removing the app's snippet/embed
  3. The merchant's internet connection
  4. Shopify automatically uninstalled the app
Answer / why predicted

Answer: B - Theme updates/replacements swap out template files; app embeds and code snippets must be re-enabled on the new theme. If the cause were servers (A), every merchant would be affected, not one. This exact cause-and-fix is the model scenario answer S1.

P2 - HIGH. "First diagnostic step" questions are the classic support-role screen; JD demands browser-console comfort.

The wishlist button is visible, but clicking it does nothing. Your FIRST diagnostic step:

  1. Reinstall the app
  2. Open the browser Console and check for JavaScript errors
  3. Ask the merchant to switch themes
  4. Escalate to engineering
Answer / why predicted

Answer: B - Cheapest, most informative first check. Reinstalling (A) is cargo-cult debugging; escalating first (D) violates the JD's "you don't escalate because something looks hard".

P3 - HIGH. Reading an unfamiliar error is literally in the JD ("read unfamiliar code without freezing").

The Console shows: Uncaught TypeError: Cannot read properties of undefined (reading 'addEventListener') thrown from theme.js, before Swym's script has loaded. Most reasonable interpretation:

  1. Swym's app has a bug
  2. A theme script is erroring — it may be breaking dependent code on the page and needs investigating first
  3. The merchant's browser is outdated
  4. The store's checkout is down
Answer / why predicted

Answer: B - The error's source file is the theme's, and it fires before the app loads. Attribute the failure to where the evidence points, then verify — don't assume the app (A) or the user's setup (C) without data.

P4 - HIGH. Device-specific bugs are daily storefront reality; tests love this discriminator.

The button shows on desktop but is missing on mobile. Most likely cause:

  1. Mobile browsers don't run JavaScript
  2. A CSS media query hides the element at small screen widths
  3. The API blocks mobile devices
  4. The phone needs to be restarted
Answer / why predicted

Answer: B - Responsive CSS (e.g., @media (max-width: 767px){ .swym-button { display:none } }) is the overwhelmingly common cause. Verify in DevTools device mode.

P5 - MEDIUM-HIGH. The incognito test is a standard support diagnostic; expect it as a question or use it as an answer.

A feature works in an incognito window but not in the merchant's normal browser. Most likely explanation:

  1. The app is fundamentally broken
  2. A browser extension (e.g., ad blocker) or stale cached files are interfering
  3. Incognito mode has higher permissions
  4. The merchant's account is suspended
Answer / why predicted

Answer: B - Incognito disables most extensions and uses a fresh cache; if behavior differs, the delta IS the extension/cache. This one-line diagnostic is worth using in your descriptive answers too.

P6 - HIGH. Back in Stock Alerts is one of Swym's three apps; "emails not arriving" is its #1 ticket.

  3. The recipient's spam folder and a test send
  4. Rewriting the email template's marketing copy

P7 - HIGH. Documented: Swym asks API questions even in non-engineering interviews.

P8 - MEDIUM. Tests the display/visibility/opacity trio at depth — a favorite discriminator, and a real "invisible button" bug.

After adding custom CSS, the wishlist icon is invisible, its space in the layout remains, and clicking that empty space still triggers the wishlist. Which property most likely caused this?

  1. display: none
  2. visibility: hidden
  3. opacity: 0
  4. position: static
Answer / quick check

Answer: C - opacity:0 = invisible + space kept + still clickable. visibility:hidden keeps space but does NOT receive clicks. display:none removes the space entirely. Three properties, three different fingerprints — memorize the triple.

P9 - HIGH. "Cannot reproduce" handling separates real support instinct from guessing; perfect SJT-technical hybrid.

  4. Escalate to engineering immediately
Answer / why predicted

P10 - HIGH. Safe-change workflow; the model answer S1 depends on it, and their interview probes exactly this judgment.

A fix requires editing the merchant's live theme code. Safest workflow:

  1. Edit the live theme directly, late at night

Answer: B - Never experiment on a live store. Duplicate → fix → preview → confirm → publish. Saying this sentence in an interview signals production-safety instinct most freshers lack.

JS gotchas, one line each

  • typeof null → "object"; typeof undefined → "undefined"; typeof NaN → "number"; NaN === NaN is false.
  • 0.1 + 0.2 === 0.3 → false (floating point).
  • Microtasks (Promises) run before macrotasks (setTimeout), after sync code.
  • var hoists as undefined; let/const hoist into the TDZ (ReferenceError before declaration).
  • const blocks reassignment, not mutation.
  • Arrow functions: no own this/arguments; can't be constructors.
  • == coerces; === doesn't. Falsy values: 0, "", null, undefined, NaN, false.
  • <script defer> runs after HTML parsing, in order; async runs whenever it arrives, order not guaranteed.
  • JSON needs double quotes; JSON.stringify drops keys whose value is undefined.

Grammar quick rules (the verbal section in 10 lines)

  • Collective nouns (team, staff, committee) → singular verb in standard test English.
  • Neither/either/each/every one + of → singular verb.
  • Since + point in time; for + duration.
  • Fewer + countable; less + uncountable.
  • Affect = verb; effect = noun (usually).
  • its = possessive; it's = it is.
  • Third conditional: If + had + V3, would have + V3 — never "if I would have".
  • Subject pronouns after "and": "my teammate and I" (subject), "with my teammate and me" (object).
  • Who = subject, whom = object; that vs which: "which" takes a comma.
  • Ban from your writing: "do the needful", "kindly revert", "the same", "prepone" — plain modern English only.
