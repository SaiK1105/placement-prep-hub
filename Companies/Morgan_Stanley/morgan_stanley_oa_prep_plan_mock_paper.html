Blind spots this plan was missing - fix these today

  • Full-program I/O, not LeetCode functions: SHL/Aspiring Minds usually gives a blank editor with stdin/stdout, no autocomplete. You're coding in Python: use sys.stdin.readline, not plain input(), which can TLE at N = 2*10^5. Template in Last-Minute Notes. Also verify Python appears in the language dropdown the moment you access the test link - if not, tell yourself today, not at 11 AM.
  • Python speed, not overflow, is your risk: ints don't overflow in Python, so ignore the long/int worry. Instead watch time limits: keep hot loops simple, and for Stock-K use the K >= N/2 greedy shortcut. Details in Last-Minute Notes.
  • Recursion limit: Python's default recursion limit is 1000 - a skewed 2*10^5-node tree crashes instantly. sys.setrecursionlimit(300000) at the top of every solution, and prefer iterative DFS for trees. Practice once today.
  • Platform rules: SHL tests are usually one-way - no going back to previous questions or sections, each section separately timed, often webcam-proctored. Answer every aptitude question before moving on; there may be no review pass. Check your invite email today for browser requirements (some links need Chrome, camera/mic permission, or a lockdown setup) - don't discover this at 10:55 AM.
  • Debugging section constraint: these platforms often restrict how much you can edit or expect a minimal fix. Change the fewest lines possible; never rewrite the whole snippet.
  • No calculator in aptitude: do the mock's math section fully on paper today. Practice percentage/ratio shortcuts (e.g., +20% then -20% = -4%) rather than long division under pressure.
  • Pseudo-code MCQs: some Morgan Stanley aptitude pools include trace-the-output pseudo-code questions. If you finish the mock early, trace 3-4 short loops by hand.
  • Logistics failure modes: re-check the invite email (and spam folder) today for the exact link and any system-check step. Keep a mobile hotspot ready as backup internet, close all other apps/notifications, use the bathroom and keep water at the desk before starting - proctored tests may not let you leave the seat.

Mock Section B - Debugging / Fix the Code

Timer: 20 minutes for 7 questions. Expected workflow: identify the bug, change the minimum number of lines, run mentally on edge cases.

Python note: even if you code Section C in Python, debugging snippets are often served in C/C++/Java and you must fix them in that language. You don't need to write Java - just read it. The bugs are language-agnostic (bounds, comparisons, base cases). The one Java-specific rule to remember: compare strings with .equals(), never ==.

Debug Q1

Reverse Sign

int reverseSign(int x) {
    if (x < 0) return x;
    return -x;
}
// Expected: reverseSign(-8) = 8, reverseSign(5) = -5
Expected fix

Change `return x;` to `return -x;` or simply return `-x` for all integer x except edge-case INT_MIN.

Debug Q2

Matrix Addition

for (int i = 0; i < rows; i++) {
    for (int j = 0; j < rows; j++) {
        C[i][j] = A[i][j] + B[i][j];
    }
}
// A, B, C have dimensions rows x cols
Expected fix

Inner loop should use `j < cols`, not `j < rows`.

Debug Q3

Bubble Sort Boundary

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n - i; j++) {
        if (arr[j] > arr[j+1]) swap(arr[j], arr[j+1]);
    }
}
Expected fix

Inner loop should be `j < n - i - 1` to avoid arr[j+1] out of bounds.

Debug Q4

Selection Sort Min Index

for (int i = 0; i < n; i++) {
    int minIndex = i;
    for (int j = i+1; j < n; j++) {
        if (arr[j] < arr[minIndex]) minIndex = i;
    }
    swap(arr[i], arr[minIndex]);
}
Expected fix

Inside the if, set `minIndex = j`, not `i`.

Debug Q5

Java String Comparison

boolean isMorgan(String s) {
    return s == "Morgan";
}
Expected fix

Use `return "Morgan".equals(s);` to compare content and avoid null issues.

Debug Q6

Factorial Base Case

long fact(int n) {
    if (n == 1) return 1;
    return n * fact(n - 1);
}
// Expected: fact(0) = 1
Expected fix

Base case should be `if (n <= 1) return 1;`.

Debug Q7

Binary Search Loop

int search(vector<int>& a, int target) {
    int l = 0, r = a.size() - 1;
    while (l < r) {
        int mid = (l + r) / 2;
        if (a[mid] == target) return mid;
        else if (a[mid] < target) l = mid + 1;
        else r = mid - 1;
    }
    return -1;
}
Expected fix

Use `while (l <= r)` so the last remaining element is checked. Prefer `l + (r-l)/2` for mid.

Coding Q1 - Maximum Non-Adjacent Bonus

Statement: You are given N project bonus values in an array. You may choose any projects, but you cannot choose two adjacent projects. Return the maximum total bonus.

Input: First line N. Second line N integers.

Output: Maximum obtainable sum.

Constraints: 1 <= N <= 2*10^5, 0 <= value[i] <= 10^9

Input: 6 / 5 5 10 100 10 5 -> Output: 110
Expected approach

DP with rolling variables: take = value[i] + dp[i-2], skip = dp[i-1]. Complexity O(N), memory O(1).

Found in public Morgan Stanley Technology Analyst experience; high-confidence DP drill.

Coding Q2 - Max Stock Profit With At Most K Transactions

Statement: Given prices for N days and an integer K, find the maximum profit possible using at most K buy-sell transactions. You must sell before buying again.

Input: First line N K. Second line N prices.

Output: Maximum profit.

Constraints: 1 <= N <= 10^5, 1 <= K <= 100, 0 <= price[i] <= 10^9

Input: 6 2 / 10 22 5 75 65 80 -> Output: 87
Expected approach

DP over transactions with best buy state. For each t, update buy[t]=max(buy[t], sell[t-1]-price), sell[t]=max(sell[t], buy[t]+price). Complexity O(NK). If K>=N/2, use unlimited-transactions greedy.

Expected from uploaded research: tree question likely; standard OA medium tree drill.

Coding Q3 - Diameter of a Binary Tree

Statement: A binary tree is given in level order with -1 representing null. Return the diameter of the tree, defined as the maximum number of edges on any path between two nodes.

Input: First line N, then N space-separated level-order values, using -1 for null.

Output: Diameter in edges.

Constraints: 0 <= N <= 2*10^5, node values fit in int

Input: 7 / 1 2 3 4 5 -1 -1 -> Output: 3 (path 4-2-1-3 or 5-2-1-3)
Expected approach

DFS returns height. At each node, answer=max(answer, leftHeight+rightHeight). Complexity O(N). Use iterative traversal if recursion depth is risky.

Last-minute notes

Coding templates to remember

  • Sliding window: maintain frequency map, move left while invalid, count windows.
  • DP rolling: only keep previous states when recurrence uses i-1/i-2.
  • Tree DFS: function returns local value; global variable stores best answer.
  • Binary search: use while (l <= r) for exact search and avoid overflow in mid.

Python recursion trap (skewed tree, N = 2*10^5)

import sys
sys.setrecursionlimit(300000)   # default is only 1000 - a chain tree crashes without this

Even with the higher limit, deep recursion can still crash on some judges. Safest: write tree DFS iteratively with an explicit stack. Practice diameter iteratively once today.

Python speed rule

Python is ~30-50x slower than C++. O(N) and O(N log N) at N = 2*10^5 are fine. O(NK) at 10^5 * 100 = 10^7 (Stock K-transactions) is borderline: keep the inner loop dead simple (plain lists, no function calls inside), and remember the shortcut - if K >= N/2, switch to the unlimited-transactions greedy which is O(N).

Common edge cases

  • Empty input / N=0 if allowed.
  • N=1 or K=0.
  • All equal values, strictly increasing, strictly decreasing.
  • Large values causing int overflow: use long/long long.
  • Tree with only one node; skewed tree causing recursion depth issues.
